offis.io python extension

the io extensions provides methods for user interaction via the automation gui

Methods

appendInfo(text)
appends a line to the user info box

clearInfo()
clears the user info box

setInfo(text)
clears the user info box and inserts thet text

alert(string text, string buttonText = "Ok")
Shortcut function to display a message using a User Request

requestYesNo(text, yes = "Yes", no = "No")
Shortcut function to request a Yes/No Choice. Returns 1 for Yes, 0 for No

startVideoRecording(int fps = 15)
Starts a video capture on display 1. This can be completed/canceled by the following methods or manually by the user. Use -1 for automatic fps detection

quitVideoRecording()
Ends the video recording on display 1 without saving the video. Ends all video recordings; issued by automation or issued manually

finishVideoRecording(string filename)
Ends the video recording on display 1 and saves the recorded video to filename. Ends all video recordings; issued by automation or issued manually

UserRequest class

Use this class to construct forms the queue input from the user

class methods

UserRequest::UserRequest(string title)
Creates a user request object with the given title

UserRequest::addString(string desc, string default = "", choices=[])
Adds a string request to the form. if choices are given, an editable drop down list will be shown

UserRequest::addInteger(string desc, int default = 0, int min = not_defined, int max = not_Defined, int step = 1)
Adds a integer request to the form. If min and max are defined, the user request must be in this range. step defines the value change if the user uses the input up/down button

UserRequest::addFloat(string desc, float default = 0.0)
Adds a float request to the form. The float can be entered by the user in scientific mode (8e-6).

UserRequest::addBool(string desc, bool default = False, string trueName = 'Yes', string falseName = 'No')
Adds a boolean request to the form. The request is displayed as radio button. The default names (Yes/No) can be changed by providing the string trueName and falseName

UserRequest::addChoice(string desc, list names, list values, mixed defaultValue = None)
Adds a drop-down list to the request. names and values must be a list of convertable types(string,int,float). Default value must match exacly one value in the values list to have an effect.

UserRequest::addInputFile(string desc, string default = "")
Adds an input file request to the form. The gui will show an input file selection button. the return value is a string

UserRequest::addOutputFile(string desc, string default = "")
Adds an output file request to the form. The gui will show an output file selection button. the return value is a string

UserRequest::addDirectory(string desc, string default = "")
Adds an directory request to the form. The gui will show a directory selection button. the return value is a string

UserRequest::addButton(string name, id = 0)
Adds a submit button to the form. The title of the button will be name. An id can be given to differentiate between buttons. The id of the selected button is returned by the execute funtion

int UserRequest::execute(string buttonText = None)
Executes the request. If button text is provided, a button with the text and the default id is added. If no buttons were added (either by addButton or by providing a button text) a default button is displayed.
The function returns the id of the pressed button

list UserRequest::values()
Returns the list of the user values. The value order represents the add[What] call sequence. If called prior to execute, an empty list is returned

UserRequest::abort()
Aborts the user request and closes the input window (for example, after a timeout)

ProgressBar class

Use this class to provide progress information to the user

class methods

ProgressBar::ProgressBar(string title = 'ProgressBar', int maximum = 100)
Creates a new progress bar with the given title. The progress is initially zero. The maximum can be changed to a different value

ProgressBar::setProgress(int progress)
Updates the progress bar to the given value. progress should be between 0 and maximum

ProgressBar::close()
Removes the progress bar form the user interface

InfoPanel class

Use this class to provide information about data to the user

class methods

InfoPanel::InfoPanel(string title = 'Info')
Creates a new InfoPanel bar with the given title. call add and show afterwards

InfoPanel::add(string key, mixed value)
Adds a line to the info Panel. Must be called before show to have effect

InfoPanel::show(string key, mixed value)
Finishes the setup of the info panel. The info panel is shown to the user. Afterwards, only update functions should be called

InfoPanel::updateAll(list values)
Updates all values of the panel. list should have the same size as the number of lines in this panel

InfoPanel::updateOne(int position, mixed value)
Updates a single value of the panel. position is the zero based index of the value (same order add was called)

InfoPanel::close()
Removes the Info panel bar form the user interface
